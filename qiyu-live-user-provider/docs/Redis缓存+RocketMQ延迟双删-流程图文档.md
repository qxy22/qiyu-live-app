### 4.2 延迟双删解决方案

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          延迟双删完整流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │                         更新用户操作                                  │ │
│    │                       updateUser(userDTO)                           │ │
│    └────────────────────────────────┬────────────────────────────────────┘ │
│                                     │                                       │
│                                     ▼                                       │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │  第一步：删除缓存（第一次删除）                                        │ │
│    │                                                                     │ │
│    │      redisTemplate.delete("qiyu:user:{userId}")                    │ │
│    │                                                                     │ │
│    │      目的：清除旧缓存，防止后续读取旧数据                              │ │
│    │                                                                     │ │
│    └────────────────────────────────┬────────────────────────────────────┘ │
│                                     │                                       │
│                                     ▼                                       │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │  第二步：更新数据库                                                   │ │
│    │                                                                     │ │
│    │      userMapper.updateById(userPo)                                 │ │
│    │                                                                     │ │
│    │      UPDATE t_user_XX SET ... WHERE user_id = ?                    │ │
│    │                                                                     │ │
│    └────────────────────────────────┬────────────────────────────────────┘ │
│                                     │                                       │
│                                     ▼                                       │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │  第三步：发送延迟消息到RocketMQ                                       │ │
│    │                                                                     │ │
│    │      MessageBuilder.withPayload(userId)                            │ │
│    │          .setHeader(DELAY_TIME_LEVEL, 1)  // 延迟1秒                │ │
│    │          .build()                                                  │ │
│    │                                                                     │ │
│    │      rocketMQTemplate.syncSend("cache-delete-topic", message)      │ │
│    │                                                                     │ │
│    └────────────────────────────────┬────────────────────────────────────┘ │
│                                     │                                       │
│                                     ▼                                       │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │                    RocketMQ 延迟队列                                  │ │
│    │                                                                     │ │
│    │      消息暂存，等待延迟时间到达                                       │ │
│    │      delayLevel = 1 → 延迟 1 秒                                     │ │
│    │                                                                     │ │
│    │      ┌─────────────────────────────────────────────┐               │ │
│    │      │  时间轴：                                    │               │ │
│    │      │  0ms   - 消息发送到Broker                    │               │ │
│    │      │  0-1000ms - 消息存储在延迟队列               │               │ │
│    │      │  1000ms - 消息投递到目标队列                 │               │ │
│    │      │  1000ms+ - 消费者拉取消息                    │               │ │
│    │      └─────────────────────────────────────────────┘               │ │
│    │                                                                     │ │
│    └────────────────────────────────┬────────────────────────────────────┘ │
│                                     │                                       │
│                                     │ 1秒后                                 │
│                                     ▼                                       │
│    ┌─────────────────────────────────────────────────────────────────────┐ │
│    │                    消费者处理消息                                     │ │
│    │                                                                     │ │
│    │      CacheDeleteConsumer.onMessage(userId)                         │ │
│    │                                                                     │ │
│    │      第四步：删除缓存（第二次删除）                                    │ │
│    │                                                                     │ │
│    │      redisTemplate.delete("qiyu:user:{userId}")                    │ │
│    │                                                                     │ │
│    │      目的：清除可能被写入的脏数据                                      │ │
│    │                                                                     │ │
│    └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
